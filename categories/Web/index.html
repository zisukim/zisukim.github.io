Kotlin and Java sources in one project

Posted on 2019-08-17 11:45 | In Web , Kotlin | Comments:

Why

  • I’m inexperienced, I don’t know. It would be a good practice if I gradually apply the developed parts in my lovely projects. 😊

Before…

  • Maven v3 is only supported.
  • Spring Boot 2.x should be with you.
  • I use..
    • Maven 3.5.0
    • Java 1.8
    • Spring boot 2.1.4

Come on !

1. Add Dependencies

  • Add the dependencies that involved in kotlin in your POM.
  • kotlin-stdlib: The standard library. If you’re targeting JDK7 or JDK8, you can use extened library. kotlin-stdlib-jdk7 or kotlin-stdlib-jdk8.
  • kotlin-reflect: Reflection for Kotlin.
  • jackson-module-kotlin: Support for serialization/deserialization of Kotlin classes.

2. Add Plugins

  • The important thing is that Kotlin sources needed to be compiled before Java sources.
  • You need to disable default-compile and default-testCompile, and set java-compile and java-test-compile of the maven-compiler-plugin.

3. Configure Kotlin with Maven

  • If you use IntelliJ IDEA, you can see the following popup.


  • Choose the modules to configure whether you want to add the runtime library.

4. Make the sample controller

  • Done and Enjoy it 😆!

References

  • [Kotlin] Using Maven - Compiling Kotlin and Java sources
  • [Spring] Spring boot - Kotlin support
  • [IntelliJ] Mixing Java and Kotlin in One Project

  • The process of the header and using a user-defined consumer is skipped. It is the same as the XLSX Sheet Handler.

It supports lots of features to handle data easily.
But when the user uploaded a large excel file that I couldn’t expect, An OOM occurred in my service. 😢
So here are some of the libraries that I used to solve that issue.

Apache POI Event API

  • Apache POI XSSF and SAX
  • It parses the Excel file by SAX. A row of the Excel is used like a node of XML.
  • It triggers events only for things actually stored. So we have to consider the position of the column.

Add the dependency (maven)

Implements the SheetContentsHandler

  • When uploading Excel, we can override and use the desired function.
  • If the hasHeader is true, the header is defined as the first line.
  • I didn’t store the whole contents (bulk rows) inside. I rely on the user’s work at the end of the row. We can define a consumer to collect a row, access a row, etc.
  • If there is no value, no event occurs. So fillEmptyCells() should be performed. You can refer to the following example about this work. XLSX2CSV.java
