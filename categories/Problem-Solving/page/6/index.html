Reverse Numbers

Posted on 2019-02-23 07:37 | In Problem Solving | Comments:

Problem

https://leetcode.com/problems/reverse-integer

  1. If input is negative, the reversed numberse is also negative.
  2. If input is out of Integer ranges, it returns 0.

Solution

Pop digits and Push them

  • To reverse numbers, we just pop last digit off and push it to the back of new values.
  • We would use mod to get units digit.
  • We have to check input is negative number or not.
  • Because we use mod to reverse input, we record input is negative or not. then make it positive number.
  • At last, We check this solution would cause an overflow.

Complexity

  • Time complexity : $O(logx)$
  • Space complexity : $O(1)$

Problem

https://leetcode.com/problems/next-closest-time/

Solution

  • In naive approach, There are only 4 * 4 * 4 * 4 possible times including duplicated numbers.
  • You just filter cases that couldn’t define as hour or minutes. For instance, It’s like 55 hours or 99 minutes.
  • And I found a clear answer that was on discussion.
    • Imagine we wrote input hour h and input minute m.
    • If there is just bigger number m1 than the input minutes(< 60), h:m1 is an answer.
    • (Else) If there is just bigger number h1 than the input hours(< 24), h1:smallest m is the next closest time.
    • (Else) The closest time is on next day.

Two Sum

Posted on 2019-01-03 13:30 | In Problem Solving | Comments:

Problem

https://leetcode.com/problems/two-sum/

Solution

  • You may assume that each input would have exactly one solution.
  • You return indices of two numbers.
  1. Brute force
  • The simplest solution is just comparing every possible pairs. The time complexity could be $O(N^2)$. Because this problem doesn’t use same element of them, it will be $N * (N-1)$.
  1. Using dictionary
  • We traverse the list containing $N$ elements twice.
  • A dictionary is built for looking up values in constant time.
  • In the first iteration, we add the value of elements as key and the index as value on dictionary.

  • Then, in the second iteration, we check if there is the number to be sum.

  • And… One loop is enough to get an answer if we care the order of outputs.

Complexity

  • Time Complexity: $O(N)$
  • Space Complexity: $O(N)$
