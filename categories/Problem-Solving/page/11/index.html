STL map, set

Posted on 2015-06-05 13:59 | In Problem Solving | Comments:

앞으로 문제를 풀 때 쓸지도 모르는 STL들 중 대표적인 2개 : map, set

map

map과 set은 거의 비슷한 자료구조형태를 가지지만
크게 다른 점 중 하나는 map은 2개의 데이터를 set은 1개의 데이터를 저장하고 있는 것이다.

map의 자료구조는 트리(Tree)
정확하게 얘기하면 이진트리
더 정확하게 얘기하면 레드블랙트리(Red-Black Tree)
(오늘은 트리 자료구조를 공부하는 시간이 아니므로 트리에 관한 설명은 스킵할게용)

우리는 map을 다음의 조건을 가질 때 사용한다.

  1. 정렬된 데이터들의 집합이 필요할 때
  2. 검색이 빠른 속도로 이루어져야할 때
  3. 삽입과 삭제가 빈번하게 일어나지 않을 때
  4. 중복된 값을 허용하지 않을 때

정렬된 데이터를 가지고 있고 검색이 vector(array)보다 빠른 이유가 트리 형태로 데이터를 가지고 있기 때문이다.
즉 map에서 우리가 찾고자 하는 값(key)을 검색하고자 할 때 vector처럼 처음부터 끝까지 뒤져야 하는 (linear time)이 아니라
이진트리의 검색 방법을 사용하여 빠른 시간 ($log n$)에 찾을 수 있다.

선언 방법은 다음과 같다
map < key의 타입, value의 타입 > m;
사전에서
“치킨”라는 단어를 찾아보니 “닭고기를튀긴것”라는 뜻을 알 수 있었다.
이 때 “치킨”는 key값이며 뜻은 “닭고기를튀긴것”이다.

데이터를 삽입하는 방법은 단순하다.
map < string, string > m; // key와 value모두 string의 형태이므로

m[“치킨”] = “닭고기를튀긴것”;

탐색하여 value값을 출력하는 것도 매우 간단하다.
cout<<m[“치킨”]<<endl;
을 하게 되면 “닭고기를튀긴것”이라는 문자열이 출력된다.

  1. 베스트셀러
    https://www.acmicpc.net/problem/1302

책의 제목이 입력된 빈도수가 가장 큰 책의 제목을 출력하면 되는 문제이다.
map 자료형을 선언하고 데이터를 삽입하는 것은 쉽게 구현했을 것이다.
최대 빈도수를 구하기 위해서는 map에 저장된 모든 컨테이너를 돌려보며 찾아야 한다.

이 때 우리는 iterator(반복자)를 사용한다.
벡터는 index를 integer형태로
for(int i = 0; i<v.size(); ++i)
로 구현할 수 있지만 맵은 조금 다른 변수를 사용하여 모든 데이터를 살펴보아야 한다.
이 문제의 경우

map < string, int > m;

map < string, int > ::iterator it; // 뒤져보고자 하는 map의 타입을 똑같이 선언해줘야한다.

for(it = m.begin(); it != m.end(); ++it)
it는 m의 맨 처음위치를 가리키고 있으며 it가 맨 끝에 올 때까지 돌릴 것이며
it++으로 인해 다음 컨테이너를 살펴볼 수 있는 것이다.
it를 주소값이라고 생각하면 이해하기 쉽다.
객체 때 배웠던 것처럼
*it를 출력하면 it가 가리키고 있는 곳의 데이터 값이 출력된다.
map의 경우 두 개의 데이터 값을 가지고 있다.
it, 즉 주소값을 이용하여 두 개의 데이터 값에 접근하는 방법은 다음과 같다

(it).first
이것은 it->first와 같다
(it).second
이것은 it->second와 같다

(예제 입력값에 의하면)
it가 “top”을 가리키는 위치에 있다면
it->first 는 “top”
it->second는 top의 빈도수인 4가 저장되어 있는 것이다.

  1. 나는야 포켓몬 마스터 이다솜
    https://www.acmicpc.net/problem/1620

다시 한번 강조해 주고 싶은 것은
map을 통해 $log n$시간에 겁색할 수 있는 값은 key값이다.
value값을 $log n$시간에 검색할 순 없다.
value값을 검색하려면 반복자를 사용하여 처음부터 끝까지 뒤져보는 방법 뿐이다.

이 문제에서 나오는 query는 문자열이기도, 숫자이기도 하다.
문자열, 숫자 모두 key값이어야 한다는 것이다.
그러므로 map 2개를 선언하여 하나는 문자열을 검색하는 데에 하나는 숫자를 검색하는 데에 사용한다.

물론 map < int, string > m1, map< string, int > m2; 로 구현해도 된다.

set

set은 map과 다르게 한 개의 데이터만 저장한다.

즉 value값이 존재하지 않고 key값만 저장하는 것이다.
한 개의 데이터를 저장하기 때문에 벡터와 비슷하게 느껴질 수 있다.
하지만 구조자체는 map과 마찬가지로 트리로 구현되어 있기 때문에
(잠시 위에서 끌어오겠심)

  1. 정렬된 데이터들의 집합이 필요할 때
  2. 검색이 빠른 속도로 이루어져야할 때
  3. 삽입과 삭제가 빈번하게 일어나지 않을 때
  4. 중복된 값을 허용하지 않을

때 사용된다.
set에 관련된 문제는 딱히 못찾아서 없심

최대 직사각형 넓이 구하기

Posted on 2015-05-28 11:42 | In Problem Solving | Comments:

Problem

https://www.acmicpc.net/problem/1725
https://algospot.com/judge/problem/read/FENCE

Solution

히스토그램과 FENCE 문제를 해결한 알고리즘을 각각 row마다 수행해주면 된다
(히스토그램은 밑변의 위치가 일정하기 때문에)

stack을 이용한 히스토그램 최대 넓이 구하기

직사각형의 넓이를 구하기 위해서 우리는 너비와 높이를 정의해주어야 한다.
내가 구하고자 하는 구간이 (i, j)일때 높이는 (i, j)의 그래프 중 가장 낮은 높이일 것이다.
즉 우리가 sweeping을 할 때 해당 구간에서 가장 낮은 값이 아닌 높이는 고려할 필요가 없다.
(앞에서 말했다시피 우리는 직사각형의 높이만 구하면 된다. 직사각형의 높이는 가장 낮은 값이다.)

솔루션은
i번째 높이가 들어왔을 때 너비 * i번째 높이를 구하고 최대 넓이를 갱신하면 된다.
너비는 left와 right를 사용하여 표현할 것이다.
이를 0부터 마지막까지 sweeping하면서 수행한다.
최대 넓이를 위한 너비는 i번째 높이보다 작은 높이들로 막혀 있다.
(i번째 높이값을 높이로 하는 직사각형이기 때문에. 더 작은 놈들은 직사각형에 포함시킬 수 없다.)

즉 왼쪽 작은 놈의 index를 left, 오른쪽 작은 놈의 index를 right라고 정의해준다.
너비는 right-left-1로써 표현될 수 있다.
left와 right의 값을 설정하기 위해 우리는 stack을 사용할 것이다.
left가 가르키는 index의 높이는 새로 들어오는 값보다 작은 값이다.
즉 새로 들어오는 값이 left가 가리키는 값보다 크다면 스택에 포함시킨다.
(갱신하지 않아도 된다. 크면 어차피 i번째 높이를 가지는 직사각형에 포함되므로)

h[st.top()] <h[new]일 경우
st.push(new)

이에 반해 새로 들어오는 값이 left가 가리키는 값보다 작거나 같으면 left를 갱신해주어야 한다.
(left는 새로 들어오는 값보다 작아야 하므로)

stack에 쌓여 있는 것들에서 new보다 작을 때까지 pop을 한다.
new보다 작은 값이 나왔다. 그러면 그것이 새로운 left가 된다.
h[st.top()]>=h[new]일 경우
st.pop(st.top())

높이는 i번째 높이를 사용한다.

재귀함수2

Posted on 2015-05-06 13:43 | In Problem Solving | Comments:

재귀함수에 대한 전반적인 이론은 재귀함수1 참조
재귀함수1에서 풀었던 두 문제 1182 부분집합의 합 4963 섬의 개수와 솔루션이 비슷한 문제들

주사위 던지기2

http://www.jungol.co.kr/bbs/board.php?bo_table=pbank&wr_id=458&sca=20

경우의 수를 더해서 조건을 만족하면 출력하는 점에서 부분집합의 합과 매우 비슷한 문제이다.
다만, 부분집합의 합과 다른 점은 다음 과 같다.

부분집합의 합

  • 재귀 : 해당 원소를 포함한 경우 하지 않는 경우
  • 인자 변화 : 배열의 index를 기반, index+1만 호출하므로 원소의 중복이 존재하지 않는다.
  • 출력 : 경우의 수

주사위 던지기2

  • 재귀 : 해당 원소를 포함한 경우(포함하지 않는 경우는 없다. 주사위는 꼭 주어진 개수만큼 던져야 하니까)
  • 인자 변화 : 사용한 주사위 개수를 기반
    (index를 기반으로 하여 index+1로 호출하였다면 중복된 개수를 셀 수 없다)
  • 출력 : 해당하는 합을 구하는 데에 사용된 원소의 목록

주사위는 최대 7개, 물론 for문 7개로 해결할 수 있다. 하지만 우린 귀찮으니 재귀로 하겠다.
최악의 경우 경우의 수는 $6^7$(279,936)개일 것이다.

함수 : n번째 주사위를 합을 구하는 데 사용하겠다.
인자 : 현재 사용한 주사위 수 n, n번째 주사위까지의 눈의 합 sum
base case : n == N+1일때 더이상 쪼개지지 않는 작업이 된다. (N개까지는 봐야하니까 N+1이다.)
내가 하려는 작업 : sum에 원소를 포함시킨다.
recursive case : n+1번째 주사위에서 내가 볼 수 있는 눈은 1~6이다.

한마디로 말해서 주사위의 개수가 3개인 경우 1 1 1, 1 1 2, 1 1 3, … 6, 6, 6까지 살펴봐야 한다.

유사문제
주사위 던지기1 http://www.jungol.co.kr/bbs/board.php?bo_table=pbank&wr_id=449&sca=20
모든 경우, 중복된 경우, 중복되지 않은 경우를 모두 다룬다

단지번호 붙이기

https://www.acmicpc.net/problem/2667

이 문제 또한 저번 시간에 풀었던 섬의 개수와 매우 유사한 문제이다.
다만 섬의 개수와 다른 점은 다음과 같다.

섬의 개수

  • component 개수 세기
  • 8방향

단지번호 붙이기

  • 한개의 component에 있는 정점들의 개수 세기
  • 4방향
    개수를 세는 연산을 어느 위치에서 해야할지 고민해보자.

함수 : map[y][x]를 방문한다.
인자 : 현재 방문할 y, x
base case : main에서 지도 끝까지 뒤졌을 때 끝난다.(함수 안에는 구현안됨)
내가 하려는 작업 : map[y][x]을 방문한다. visited[y][x]에 방문했다는 표시를 해준다.
recursive case : 4방향을 살펴본 뒤 조건에 맞으면 다음 y, x인자를 설정하여 함수를 호출한다.

유의할 점 : input이 빈칸 없는 1, 0으로 이루어 진 것을 주의하자
