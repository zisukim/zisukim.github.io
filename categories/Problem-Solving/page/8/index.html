쉬운 계단 수

Posted on 2015-10-20 13:13 | In Problem Solving | Comments:

Problem

https://www.acmicpc.net/problem/10844

Solution

느낀 점1 : dp미지수를 한개만 가지고 만들 때 뭔가 오묘하고 확실치 않다면 두개를 만들면 된다.

저번주 시험에서도 느꼈는데 dp를 반드시 미지수 1개, 2개만 생각하다 보니까 …. ㅜㅜ 쉬운 문제를.

dp[i] : i자리 수 중에서 계단 수의 갯수

이러니까 1~8은 두 개씩 추가되지만 0과 9는 1개씩 추가되므로 무작정 두 배를 할 수가 없었다.
0과 9의 갯수를 계산하기보다 바로 미지수 하나 더 추가해서 dp를 구현하면 된다.

느낀 점2 : 대충 대충 손 먼저 움직이지 말고1. 기저사례 2. 다음조건 확실히 정의해서 짜자.

배낭 채우기 (Knapsack Problem)

Posted on 2015-10-17 20:11 | In Problem Solving | Comments:

Problem

http://www.jungol.co.kr/bbs/board.php?bo_table=pbank&wr_id=357&sca=3050

Solution

유명한 Knapsack Problem

배낭 채우기 문제
배낭에 N개의 보물을 배낭에 담으려고 한다.
각각 보물은 weight[i]의 무게를 가지고 p[i]의 가치를 가지고 있다.
총 보물의 무게가 W까지만 들어간다고 할 때 들어간 보물들의 가치가 최대가 되도록 보물을 담는 것이다.
그 경우 들어간 보물들의 최대 가치를 구하는 문제이다.
보물의 갯수는 제한이 없다.

이 문제를 풀 때 그리디방법과 dp방법이 있는데
그리디는 오답의 확률이 있으므로 dp로 푸는 것이 옳다.
(그리디 방법은 대충 단위 무게당 가치가 큰 것들을 우선으로 넣고,
더이상 들어가지 않으면 더 작은 것을 넣는 방식으로 구현한다.)

dp는 이차원 형태로 정의한다. (나타내는 미지수가 두 개이기 때문에)
