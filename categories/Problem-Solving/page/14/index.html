Short Circuit Evaluation

조건이 두개가 있을 때 첫번째 조건을 만족하면 두번째 조건은 거들떠 보지도 않고 true/false값을 결정하는거
A && B 일때 A가 false이면 B는 보지도 않고 false, A || B 일때 A가 true이면 B는 보지도 않고 true
C관련 언어에서 지원 ㅇㅇ

예전에 2차배열에서 dfs할때 범위를 나타낼때 bool inRange(y, x)로 나타냈는데
y랑 x가 범위밖의 값을 나타낼 때
if(inRange(y, x) && arr[y][x]) 이러면 걍 false로 끝나지만(배열 접근 안함)
if(arr[y][x] && inRange(y, x)) 이러면 배열에 접근해서 error남ㅎㅎㅎㅎ

lower_bound()

Posted on 2014-11-06 00:33 | In Problem Solving | Comments:

vector v;
vector ::iterator it;

it=lower_bound(v.begin(), v.end(), val);
수행시간은… 뭘까…ㅋㅋㅋㅋㅋ

val와 같거나 큰 숫자 중 가장 작은 수의 위치를 리턴하는 함수
그런데 가장 작은 수가 여러 개가 있다면?
중복된 수 중 가장 처음 위치를 가리킴 !
그니까 문제 “방사능”의 경우 r이 625일때 lower_bound()의 val를 625로 해주면 만약 625가 여러개가 있을 경우에도 1개 밖에 count되지 않는다
625+1로 val를 설정해주면 625인 중복된 값도 모두 셀 수 있다

upper_bound는 같거나 작은 수 중 가장 큰 수의 위치를 리턴해주므로 625를 그대로 val를 해줘도 됨
중복된 값중 가장 마지막 위치를 리턴하므로

문자열 입력

Posted on 2014-10-10 20:20 | In Problem Solving | Comments:

문자열 중에 space가 있을때

gets(str)

EOF도달했을때 NULL return;
입력받은 문자들을 문자열로 만들어주고 맨 마지막에 0 넣어줌
원래 지정된 str길이보다 긴 문자열이 들어오면 overflow

fgets(str, SIZE, stdin/stdout)

EOF도달했을때 NULL return;
특정 stream으로 문자열을 입력받아 SIZE만큼 str의 시작부분부터 저장
문자열을 다 입력한 다음 enter도 들어가고 난리 남
SIZE를 지정해주니까 overflow가 날 위험은 적지만 buffer주의!
fgets()로 stream을 통해 데이터를 받을 때 바로 받는게 아니라 buffer로 받고 거기서
SIZE만큼 가져오는 거 나머지 문자는 여전히 buffer에 있음!

BFS에서 최단 경로 후보들 찾기

Posted on 2014-09-23 00:37 | In Problem Solving | Comments:

parent 배열도 있지만 얘는 경로 중복되도 한놈밖에 안나오므로

시작점 S -> 도착점 D일때 (단방향 u->v)일시
adj1 : u->v , S에서 출발하여 dist1
adj2 : v->u , D에서 출발하여 dist2

dist1 + dist2 == 최단거리 일때 당연히 얘네는 최단경로의 후보가 됨!

미로 탐색

Posted on 2014-09-13 19:17 | In Problem Solving | Comments:

Problem

https://www.acmicpc.net/problem/2178

Solution

되게 당연한건데 ㅋㅋㅋㅋ
좌표상의 BFS는 평소 다루었던 V, E가 있는 그래프랑 똑같다

101111
101010
101011
111011

여기서 1로 된 곳만 지나갈 수 있다고 했을 때 어떻게 할까
각각의 좌표 자체를 V로 본다. 그리고 인접한 위치에 있는 1에 이동할 수 있다고 생각하면 됨
그니까 여기서(0,0)이라는 정점과 (1,0)이라는 정점은 간선으로 연결되어 있는검

아 또
이런 좌표를 받을 때 (띄어쓰기 없는) int형 배열로 바로 받았다가는 배열 하나에 101111이 들어감
string으로 받고 다시 int에다가 이쁘게 구축해주면 됨
