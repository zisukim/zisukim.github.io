Kotlin?

Veröffentlicht am 2018-04-10 09:19 | in Web . Kotlin | Kommentare:

Kotlin은 Statically typed 언어이다.

Statically typed

프로그램의 모든 표현식 유형이 컴파일 타임에 알려진다.
컴파일러는 액세스하려는 메소드와 필드가 사용중인 객체에 있는지 확인할 수 있다.

Dynamically typed

런타임에 메서드 및 필드 참조를 해결할 수있는 변수 및 함수를 정의 할 수 있다.
더 짧은 코드와보다 유연한 데이터 구조 생성이 가능하다.

코틀린은 type inference를 지원하기 때문에 타입을 명시적으로 선언할 필요가 없다.

Kotlin은 함수형이고 객체 지향이다.

함수형 프로그래밍

First class functions
Immutability
No side effects

함수형은 뭐가 좋음?

conciseness
safe multithreading
easier testing

Link

  • The data type, Dictionary represents the implementation of hash tables in Python.
