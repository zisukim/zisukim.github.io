구명보트

Posted on 2019-06-08 19:31 | In Problem Solving | Comments:

Problem

https://programmers.co.kr/learn/courses/30/lessons/42885

Solution

  • The important thing is that we can put only two people in the lifeboat at a time.
  • The closer the sum of A and B is to the limit, the less lifeboats can be used.
  • If there is 40kg person, a 70kg person should ride it than a 50kg person.
    So the sum of two will be closer to the limit. It means that we have to try the heavier one first.
  • If you want to see the solution you implemented with C++, See the following link.
    https://gaussian37.github.io/interview-ps-p42885/

Complexity

  • Time Complexity: $O(nlogn)$ (for sorting)
  • Space Complexity: $O(1)$

Solution

  • If you solve it using brute force, It takes $O(n^2)$ time complexity.
    The way is to select two different numbers and compare the profits.
  • We just need the minimum price and the maximum price. If we manage the minimum price, The only the one loop is required.

Complexity

  • Time Complexity: $O(n)$
  • Space Complexity: $O(1)$
