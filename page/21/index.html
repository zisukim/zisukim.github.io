MySQL 특정 단어가 포함된 레코드 검색 (LIKE)

Posted on 2016-08-09 00:30 | In ETC | Comments:

특정 단어가 포함된 레코드를 뽑아내고 싶을 때, 우리는 다양한 방법을 사용할 수 있다.
그 중 가장 간단하게 사용하는 LIKE 에 대해 알아보도록 하자.

SQL 패턴 매칭

MySQL에서는 정규표현식뿐만 아니라 이 SQL 패턴 매칭을 지원하고 있다.

문자 뜻 예제
_ 한개의 문자 매칭 SELECT * FROM user WHERE name LIKE ‘chicke_’;
% 정해지지 않은 여러 문자 매칭 SELECT * FROM user WHERE name LIKE ‘chi%’;

SQL 패턴매칭은 대소문자 구분이 없고, =, <> 대신 LIKE, NOT LIKE를 사용한다.

mybatis, ibatis에서 LIKE는 어떻게 사용할까?

실제로 써본 것은 mybatis밖에 없지만 ..!
mybatis와 ibatis는 LIKE 검색의 ‘%’을 인식하지 못한다.

(이딴게 될일이 없다는 것..)

ibatis

mybatis

concat()은 뭔가요

2개이상의 문자열을 연결하는 (concatenate) 함수이다.

그러니까 CONCAT(#{search_value}, ‘%’) 라는 것은 #{search_value}와 매핑되는 문자열 뒤에 ‘%’ 이놈이 연결되는 것이다. (PreparedStatement)

LIKE ‘${search_value}%’ 는요?

위의 concat() 방법 외에도 ‘${search_value}%’ SQL 패턴 매칭을 나타낼 수 있다.

이것은 ${value}와 #{value}와의 차이인데..!

PreparedStatement (#{value})

Mybatis로 하여금 PreparedStatement 프로퍼티를 만들어서 PreparedStatement 파라미터 (?)에 값을 셋팅하게 한다.
당연히 ‘?%’ 이런식으로 던져지므로 CONCAT으로 문자열을 이어져야 한다.

String substition(${value})

SQL 구문에 변하지 않는 값으로 넣는 빠른 방법이다.
SQL에 던져질 때 ‘김%’과 같이 문자열이 변환되어 들어간다.

하지만, 사용자로부터 받은 값을 변경하지 않고 바로 던지는 것은 안전하지 않다.
(아시다시피 SQL 주입 공격에 노출되기 때문이다. CONCAT으로 쓰세염…)

서로 다른 자연수의 합

Posted on 2016-02-16 18:55 | In Problem Solving | Comments:

Problem

https://www.acmicpc.net/problem/9764

Solution

  1. 1+1+2와 같은 중복된 수 더하기는 카운트에서 제외, 2+3와 3+2는 같으므로 1만 카운트한다.

  2. 전형적인 DP문제로 선택했다 안했다 방법으로 풀이할 수 있다.

  3. 기본적인 개념은 다음과 같다.

서로 다른 자연수들을 더해 5라는 숫자를 만들고자 한다.
5를 만드는 경우의 수 = 1이 포함된 경우 4를 만드는 경우의 수 + 1이 포함되지 않은 경우 4를 만드는 경우의 수
위와 같이 나타낼 수 있을 것이다.
뭐 1이 포함된 경우 5를 만드는 경우의 수는 1 (1+4)
1이 포함되지 않는 경우 4를 만드는 경우의 수 2 (2+3) (5)
총 3가지가 나오게 되는 것이다.

  1. 방법이 이해가 되었다면 DP테이블을 정의해보자. 미지수가 2개이므로 2차 행렬을 사용하였다.

  2. 재귀로 표현해 보면
    solution(num-used, used+1) : used를 포함한 경우
    solution(num, used+1) : used를 포함하지 않는 경우

우왕 ~

  1. 물론 재귀 말고 for문으로도 표현할 수 있다. 역시 기본 개념은 같다. 다시 말하면 ‘기존의 값 + used = 새로운 값’의 방법

  2. 코드를 먼저 보자.

  3. 넘나 짧아짐..ㅎㅎ 동작하는 것을 따라가보자.

  4. 여튼 엄청나게 전형적인 DP문제였다. 끝!

Complexity

  • Time complexity : $O(N^2)$
  • Space complexity : $O(N)$

최대 정사각형

Posted on 2016-02-16 17:32 | In Problem Solving | Comments:

Problem

https://www.acmicpc.net/problem/4095

Solution

  1. 주어진 행렬은 0과 1로만 이루어져 있다.
  2. arr[i][j] = 1이면 길이가 1인 정사각형이 이루어 진다.
  3. 정사각형의 길이를 확장시켜보자. 길이 2인 정사각형이 형성되려면 arr[i][j], arr[i-1][j], arr[i][j-1], arr[i-1][j-1], 4개의 칸이 모두 1이어야 한다.
  4. 길이 3, 4, … n인 정사각형이 형성되려면 3번처럼 계속해서 만족시켜야 할 것이다.

이 사실을 그때마다 arr[i-n][j], arr[i-n+1][j]….. 등등 n X n의 배열을 각각 확인해 보기에는 많은 시간이 걸린다.

  1. DP를 사용하여 arr[i][j]의 위치에서 만들 수 있는 최대 정사각형 변의 길이를 저장해보자.
  2. arr[i][j] = 0인 경우 정사각형이 만들어지지 않으므로 d[i][j]에 저장할 값은 없다.
  3. arr[i][j] = 1인 경우 정사각형을 확장할 기회가 주어진다.
    (다시 길이 2인 정사각형을 생각해보자. 만약 arr[i-1][j], arr[i][j-1], arr[i-1][j-1]의 값이 모두 1이였다면 길이 1인 정사각형에서 길이 2인 정사각형으로 확장될 수 있다.)

이런 작업을 하기 위해서 우리가 필요로 하는 값은 무엇일까?
우리가 필요한 값을 dp테이블에 저장해보자.

  1. DP테이블을 정의하자.
