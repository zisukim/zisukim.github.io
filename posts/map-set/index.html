STL map, set

Posted on 2015-06-05 13:59 | In Problem Solving | Comments:

앞으로 문제를 풀 때 쓸지도 모르는 STL들 중 대표적인 2개 : map, set

map

map과 set은 거의 비슷한 자료구조형태를 가지지만
크게 다른 점 중 하나는 map은 2개의 데이터를 set은 1개의 데이터를 저장하고 있는 것이다.

map의 자료구조는 트리(Tree)
정확하게 얘기하면 이진트리
더 정확하게 얘기하면 레드블랙트리(Red-Black Tree)
(오늘은 트리 자료구조를 공부하는 시간이 아니므로 트리에 관한 설명은 스킵할게용)

우리는 map을 다음의 조건을 가질 때 사용한다.

  1. 정렬된 데이터들의 집합이 필요할 때
  2. 검색이 빠른 속도로 이루어져야할 때
  3. 삽입과 삭제가 빈번하게 일어나지 않을 때
  4. 중복된 값을 허용하지 않을 때

정렬된 데이터를 가지고 있고 검색이 vector(array)보다 빠른 이유가 트리 형태로 데이터를 가지고 있기 때문이다.
즉 map에서 우리가 찾고자 하는 값(key)을 검색하고자 할 때 vector처럼 처음부터 끝까지 뒤져야 하는 (linear time)이 아니라
이진트리의 검색 방법을 사용하여 빠른 시간 ($log n$)에 찾을 수 있다.

선언 방법은 다음과 같다
map < key의 타입, value의 타입 > m;
사전에서
“치킨”라는 단어를 찾아보니 “닭고기를튀긴것”라는 뜻을 알 수 있었다.
이 때 “치킨”는 key값이며 뜻은 “닭고기를튀긴것”이다.

데이터를 삽입하는 방법은 단순하다.
map < string, string > m; // key와 value모두 string의 형태이므로

m[“치킨”] = “닭고기를튀긴것”;

탐색하여 value값을 출력하는 것도 매우 간단하다.
cout<<m[“치킨”]<<endl;
을 하게 되면 “닭고기를튀긴것”이라는 문자열이 출력된다.

  1. 베스트셀러
    https://www.acmicpc.net/problem/1302

책의 제목이 입력된 빈도수가 가장 큰 책의 제목을 출력하면 되는 문제이다.
map 자료형을 선언하고 데이터를 삽입하는 것은 쉽게 구현했을 것이다.
최대 빈도수를 구하기 위해서는 map에 저장된 모든 컨테이너를 돌려보며 찾아야 한다.

이 때 우리는 iterator(반복자)를 사용한다.
벡터는 index를 integer형태로
for(int i = 0; i<v.size(); ++i)
로 구현할 수 있지만 맵은 조금 다른 변수를 사용하여 모든 데이터를 살펴보아야 한다.
이 문제의 경우

map < string, int > m;

map < string, int > ::iterator it; // 뒤져보고자 하는 map의 타입을 똑같이 선언해줘야한다.

for(it = m.begin(); it != m.end(); ++it)
it는 m의 맨 처음위치를 가리키고 있으며 it가 맨 끝에 올 때까지 돌릴 것이며
it++으로 인해 다음 컨테이너를 살펴볼 수 있는 것이다.
it를 주소값이라고 생각하면 이해하기 쉽다.
객체 때 배웠던 것처럼
*it를 출력하면 it가 가리키고 있는 곳의 데이터 값이 출력된다.
map의 경우 두 개의 데이터 값을 가지고 있다.
it, 즉 주소값을 이용하여 두 개의 데이터 값에 접근하는 방법은 다음과 같다

(it).first
이것은 it->first와 같다
(it).second
이것은 it->second와 같다

(예제 입력값에 의하면)
it가 “top”을 가리키는 위치에 있다면
it->first 는 “top”
it->second는 top의 빈도수인 4가 저장되어 있는 것이다.

  1. 나는야 포켓몬 마스터 이다솜
    https://www.acmicpc.net/problem/1620

다시 한번 강조해 주고 싶은 것은
map을 통해 $log n$시간에 겁색할 수 있는 값은 key값이다.
value값을 $log n$시간에 검색할 순 없다.
value값을 검색하려면 반복자를 사용하여 처음부터 끝까지 뒤져보는 방법 뿐이다.

이 문제에서 나오는 query는 문자열이기도, 숫자이기도 하다.
문자열, 숫자 모두 key값이어야 한다는 것이다.
그러므로 map 2개를 선언하여 하나는 문자열을 검색하는 데에 하나는 숫자를 검색하는 데에 사용한다.

물론 map < int, string > m1, map< string, int > m2; 로 구현해도 된다.

set

set은 map과 다르게 한 개의 데이터만 저장한다.

즉 value값이 존재하지 않고 key값만 저장하는 것이다.
한 개의 데이터를 저장하기 때문에 벡터와 비슷하게 느껴질 수 있다.
하지만 구조자체는 map과 마찬가지로 트리로 구현되어 있기 때문에
(잠시 위에서 끌어오겠심)

  1. 정렬된 데이터들의 집합이 필요할 때
  2. 검색이 빠른 속도로 이루어져야할 때
  3. 삽입과 삭제가 빈번하게 일어나지 않을 때
  4. 중복된 값을 허용하지 않을

때 사용된다.
set에 관련된 문제는 딱히 못찾아서 없심
