재귀함수2

Posted on 2015-05-06 13:43 | In Problem Solving | Comments:

재귀함수에 대한 전반적인 이론은 재귀함수1 참조
재귀함수1에서 풀었던 두 문제 1182 부분집합의 합 4963 섬의 개수와 솔루션이 비슷한 문제들

주사위 던지기2

http://www.jungol.co.kr/bbs/board.php?bo_table=pbank&wr_id=458&sca=20

경우의 수를 더해서 조건을 만족하면 출력하는 점에서 부분집합의 합과 매우 비슷한 문제이다.
다만, 부분집합의 합과 다른 점은 다음 과 같다.

부분집합의 합

  • 재귀 : 해당 원소를 포함한 경우 하지 않는 경우
  • 인자 변화 : 배열의 index를 기반, index+1만 호출하므로 원소의 중복이 존재하지 않는다.
  • 출력 : 경우의 수

주사위 던지기2

  • 재귀 : 해당 원소를 포함한 경우(포함하지 않는 경우는 없다. 주사위는 꼭 주어진 개수만큼 던져야 하니까)
  • 인자 변화 : 사용한 주사위 개수를 기반
    (index를 기반으로 하여 index+1로 호출하였다면 중복된 개수를 셀 수 없다)
  • 출력 : 해당하는 합을 구하는 데에 사용된 원소의 목록

주사위는 최대 7개, 물론 for문 7개로 해결할 수 있다. 하지만 우린 귀찮으니 재귀로 하겠다.
최악의 경우 경우의 수는 $6^7$(279,936)개일 것이다.

함수 : n번째 주사위를 합을 구하는 데 사용하겠다.
인자 : 현재 사용한 주사위 수 n, n번째 주사위까지의 눈의 합 sum
base case : n == N+1일때 더이상 쪼개지지 않는 작업이 된다. (N개까지는 봐야하니까 N+1이다.)
내가 하려는 작업 : sum에 원소를 포함시킨다.
recursive case : n+1번째 주사위에서 내가 볼 수 있는 눈은 1~6이다.

한마디로 말해서 주사위의 개수가 3개인 경우 1 1 1, 1 1 2, 1 1 3, … 6, 6, 6까지 살펴봐야 한다.

유사문제
주사위 던지기1 http://www.jungol.co.kr/bbs/board.php?bo_table=pbank&wr_id=449&sca=20
모든 경우, 중복된 경우, 중복되지 않은 경우를 모두 다룬다

단지번호 붙이기

https://www.acmicpc.net/problem/2667

이 문제 또한 저번 시간에 풀었던 섬의 개수와 매우 유사한 문제이다.
다만 섬의 개수와 다른 점은 다음과 같다.

섬의 개수

  • component 개수 세기
  • 8방향

단지번호 붙이기

  • 한개의 component에 있는 정점들의 개수 세기
  • 4방향
    개수를 세는 연산을 어느 위치에서 해야할지 고민해보자.

함수 : map[y][x]를 방문한다.
인자 : 현재 방문할 y, x
base case : main에서 지도 끝까지 뒤졌을 때 끝난다.(함수 안에는 구현안됨)
내가 하려는 작업 : map[y][x]을 방문한다. visited[y][x]에 방문했다는 표시를 해준다.
recursive case : 4방향을 살펴본 뒤 조건에 맞으면 다음 y, x인자를 설정하여 함수를 호출한다.

유의할 점 : input이 빈칸 없는 1, 0으로 이루어 진 것을 주의하자

유사문제
유기농 배추 https://www.acmicpc.net/problem/1012
영역 구하기 https://www.acmicpc.net/problem/2583
